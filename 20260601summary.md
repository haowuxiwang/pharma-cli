## 一、项目目标

创建一个 AI-agent 友好的统计分析 CLI 工具，覆盖 ~90% 的 Minitab/JMP 功能，包括：
- SPC (统计过程控制)
- DOE (实验设计)
- 正态性检验
- 假设检验
- 回归分析
- 能力分析

**架构设计**:
- CLI 层：面向 AI Agent，JSON 输入输出
- R 引擎层：通过 subprocess 调用 Rscript
- 可选 GUI 层：面向人类用户（待实现）

### 3.2 遇到并修复的问题

| 问题 | 原因 | 修复方式 |
|------|------|----------|
| Minitab COM 失败 | Minitab 15 太旧，COM 未注册 | 放弃 Minitab，改用 R |
| Rscript stdin segfault | bash heredoc 传递 R 代码导致段错误 | 改用临时 .R 文件执行 |
| Anderson-Darling 小样本报错 | `ad.test()` 要求 n > 7 | 添加样本量判断，小样本返回 NA |
| ANOVA 找不到 'value' 对象 | JSON 二维数组被解析为 matrix 而非 list | 添加 `is.matrix()` 判断，按列提取 |
| TukeyHSD 访问错误 | `tukey$group` 应为 `tukey[[1]]` | 修正访问方式 |
| regression fitted_values 报错 | `$fittedvalues` 应为 `$fitted.values` | 修正属性名 |

## 四、潜在问题

### 4.1 已知限制

1. **小样本限制**:
   - Anderson-Darling 检验要求 n > 7
   - Dixon 检验要求 n ≤ 30
   - 部分控制图需要足够子组数据

2. **R 引擎依赖**:
   - 必须安装 R 且路径正确
   - R 包必须预先安装
   - subprocess 调用有 60 秒超时限制

3. **CLI 参数格式**:
   - `--x` 和 `--y` 使用 `multiple=True`，需要重复参数：`--x 1 --x 2 --x 3`
   - `-g` 参数接受 JSON 数组字符串：`-g '[1,2,3]'`
   - 文件输入支持 .txt (每行一个值)、.csv、.json

4. **未测试的命令**:
   - `correlation` - 未实际测试
   - `trend` - 未实际测试
   - `doe` - 未实际测试

### 4.2 潜在风险

1. **R 脚本错误处理**:
   - 部分 R 脚本可能对边界情况处理不完善
   - 某些检验在极端数据下可能失败

2. **JSON 序列化**:
   - R 的 `NA` 转换为 JSON 的 `null`，部分场景可能需要特殊处理
   - 大数据集可能受内存限制

3. **跨平台兼容**:
   - 路径硬编码为 Windows 格式 (`D:\R-4.6.0\bin`)
   - Linux/Mac 需要修改 `RSCRIPT_PATH` 环境变量

### 7.3 AI Agent 集成

```python
import subprocess
import json

def run_pharma_cli(command, data):
    """调用 pharma-cli 并返回结果"""
    result = subprocess.run(
        ["python", "-m", "cli", command],
        input=json.dumps(data),
        capture_output=True,
        text=True
    )
    return json.loads(result.stdout)

# 示例：描述性统计
data = {"values": [10.2, 10.5, 10.1, 10.3, 10.4]}
result = run_pharma_cli("descriptive", data)
print(result)
```
